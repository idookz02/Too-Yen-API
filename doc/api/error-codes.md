# Error Code Reference

Every error response uses the envelope from [README.md](README.md#conventions):

```json
{ "error": { "code": "RECIPE_NOT_FOUND", "message": "Recipe not found" } }
```

`INCOMPLETE_RECIPE` additionally carries a `details` array (missing field names). This list is generated by reading every `throw` site in `src/` (services, controllers, shared plugins) — not hand-maintained from memory — so it reflects what the server actually sends. Regenerate by re-grepping `badRequest|unauthorized|forbidden|notFound|conflict|new AppError` under `src/` after adding a new error.

## Cross-cutting (any endpoint)

| Status | Code | Source | When |
|---|---|---|---|
| 400 | `VALIDATION_ERROR` | Elysia TypeBox validation | Request body/query/params fail schema (wrong type, missing required field, bad file magic bytes) |
| 401 | `UNAUTHENTICATED` | [auth.plugin.ts](../../src/shared/plugins/auth.plugin.ts) | Missing/invalid/expired Bearer token on a protected route |
| 403 | `FORBIDDEN` | [auth.plugin.ts](../../src/shared/plugins/auth.plugin.ts) | `role !== 'admin'` on an admin-only route |
| 404 | `NOT_FOUND` | Elysia router | Path doesn't match any route |
| 429 | `RATE_LIMITED` | [rate-limit.plugin.ts](../../src/shared/plugins/rate-limit.plugin.ts) | Over the per-IP limit on `/auth/*` (`RATE_LIMIT_AUTH_MAX`, default 10/min) or `/search/by-image` (`RATE_LIMIT_IMAGE_SEARCH_MAX`, default 5/min) |
| 500 | `INTERNAL_ERROR` | global `onError` | Unhandled exception — treat as a bug, not a condition to branch on |

## Auth — `/auth/*`

| Status | Code | Endpoint | When |
|---|---|---|---|
| 409 | `DUPLICATE_ACCOUNT` | POST `/auth/signup` | Email or username already taken |
| 401 | `INVALID_CREDENTIALS` | POST `/auth/login` | Wrong username or password |
| 404 | `ACCOUNT_NOT_FOUND` | POST `/auth/forgot-password/check` | `identifier` (must be an email) has no matching account |
| 400 | `PASSWORD_MISMATCH` | POST `/auth/forgot-password/reset` | `new_password` and `confirm_password` differ |
| 401 | `INVALID_RESET_TOKEN` | POST `/auth/forgot-password/reset` | Reset token bad, expired, or already used |

## Profile — `/users/me/*`

| Status | Code | Endpoint | When |
|---|---|---|---|
| 404 | `USER_NOT_FOUND` | GET/PATCH `/users/me` | Token's user no longer exists (deleted account) |
| 400 | `PASSWORD_POLICY_VIOLATION` | PATCH `/users/me` | New password shorter than the minimum length |
| 400 | `FILE_TOO_LARGE` | PUT `/users/me/avatar` | Upload exceeds the 5 MB image input cap |

## Recipes — `/recipes/*`

| Status | Code | Endpoint | When |
|---|---|---|---|
| 400 | `VALIDATION_ERROR` | POST/PATCH `/recipes` | Wrong content-type (must be multipart), `data` isn't valid JSON against the recipe schema, `cover` or a step's `image_field` part isn't an image (or `image_field` is missing/reserved), duplicate `step_number`, an ingredient/equipment sent with neither its `*_id` nor a `name`, a not-found `ingredient_id`/`unit_id`/`equipment_id`, or `publish` sent on PATCH |
| 404 | `RECIPE_NOT_FOUND` | GET/PATCH/DELETE `/recipes/{id}`, publish, visibility | No recipe with that id |
| 403 | `FORBIDDEN` | GET `/recipes/{id}` (draft/private by non-owner), PATCH/DELETE (non-owner) | Caller isn't the recipe owner |
| 400 | `INCOMPLETE_RECIPE` | POST `/recipes/{id}/publish`, PATCH `/recipes/{id}` (would break a published recipe), DELETE media (removing the cover) | AC M2-1 checklist fails — `details` array lists the missing fields |
| 400 | `INVALID_STATUS` | POST `/recipes/{id}/publish` | Recipe isn't currently `draft` |
| 400 | `VALIDATION_ERROR` | PATCH `/recipes/{id}/visibility` | Target status isn't `published`/`private`, or current status is `draft` |
| 400 | `VALIDATION_ERROR` | POST `/recipes/{id}/media` | File content-type doesn't match the declared `type` |
| 409 | `VIDEO_LIMIT` | POST `/recipes/{id}/media` | Recipe already has a video (max one) |
| 404 | `MEDIA_NOT_FOUND` | DELETE `/recipes/{id}/media/{mediaId}` | No media with that id on that recipe |
| 404 | `STEP_NOT_FOUND` | PUT `/recipes/{id}/steps/{stepNumber}/image` | No step with that number on the recipe |
| 400 | `FILE_TOO_LARGE` | POST `/recipes/{id}/media`, PUT `.../steps/{n}/image` | Image > 5 MB or video > 50 MB input cap |

## Engagement — `/recipes/{id}/{like,favorite,comments}`, `/comments/{id}`

| Status | Code | Endpoint | When |
|---|---|---|---|
| 404 | `RECIPE_NOT_FOUND` | like/favorite/comments on `/recipes/{id}` | No recipe with that id |
| 403 | `FORBIDDEN` | like/favorite/comment on a non-published recipe | Engagement is published-recipes-only (decision 2026-07-10) |
| 400 | `VALIDATION_ERROR` | PATCH `/comments/{id}` | Both `image` and `remove_image=true` sent (mutually exclusive) |
| 404 | `COMMENT_NOT_FOUND` | PATCH/DELETE `/comments/{id}` | No comment with that id (or already soft-deleted) |
| 403 | `FORBIDDEN` | PATCH/DELETE `/comments/{id}` | Caller isn't the comment owner (ADR-008 — post owners can't delete others' comments) |
| 400 | `FILE_TOO_LARGE` | POST `/recipes/{id}/comments`, PATCH `/comments/{id}` | Comment image > 5 MB |

## Search — `/search/*`, `/ingredients`, `/units`

| Status | Code | Endpoint | When |
|---|---|---|---|
| 400 | `VALIDATION_ERROR` | GET `/search/match` | Neither `ingredient_ids` nor `equipment_ids` provided |
| 503 | `FEATURE_DISABLED` | POST `/search/by-image` | Server has no `OPENAI_API_KEY` configured |
| 502 | `VISION_API_ERROR` | POST `/search/by-image` | OpenAI call failed, or returned empty/malformed content |
| 429 | `RATE_LIMITED` | POST `/search/by-image` | Over `RATE_LIMIT_IMAGE_SEARCH_MAX` (default 5/min per IP) |
| 400 | `FILE_TOO_LARGE` | POST `/search/by-image` | Image > 5 MB input cap |

## Admin Master — `/admin/master/*` (admin only)

| Status | Code | Endpoint | When |
|---|---|---|---|
| 403 | `FORBIDDEN` | every route in this module | Caller's `role !== 'admin'` |
| 400 | `VALIDATION_ERROR` | POST/PATCH `/admin/master/{type}` | `min_likes` missing for `type=tiers` |
| 409 | `DUPLICATE_ENTRY` | POST/PATCH `/admin/master/{type}` | `name` (or `min_likes` for tiers) already exists among active entries — per ADR-003, creating with a name that matches an *inactive* row reactivates it instead of erroring |
| 404 | `ENTRY_NOT_FOUND` | PATCH/DELETE `/admin/master/{type}/{id}` | No entry with that id for that type |

## Masters — `/master/{type}` (public)

No auth, no domain error codes beyond the cross-cutting `VALIDATION_ERROR` (unknown `type`) and `NOT_FOUND`.
